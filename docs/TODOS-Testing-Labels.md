# GitHub Labels for Testing and Project Management

Labels help organize issues and make it easier to track different types of work, especially when you'll have multiple issues for testing, migration, features, etc.

### Testing Labels:

- `unit-tests` - Unit test implementation (color: `#0075ca` - blue)
- `integration-tests` - DOM and integration tests (color: `#1d76db` - darker blue)
- `e2e-tests` - End-to-end tests (color: `#0e8a16` - green)
- `test-coverage` - Coverage-related tasks (color: `#fbca04` - yellow)

### Project Phase Labels:

- `typescript-migration` - TypeScript conversion tasks (color: `#5319e7` - purple)
- `copilot-agent` - For testing the coding agent (color: `#d73a4a` - red)
- `documentation` - Documentation updates (color: `#0075ca` - blue)

### Priority Labels:

- `priority: high` - Important tasks (color: `#d93f0b` - orange)
- `priority: low` - Nice to have (color: `#c5def5` - light blue)

### Status Labels:

- `in-progress` - Currently being worked on (color: `#fbca04` - yellow)
- `needs-review` - Ready for review (color: `#0e8a16` - green)
